# 📌 未排程数量存放 - 快速答案

## 问题
**当用户批排程时，选择排60卷(总共100卷)，剩余40卷应该存放在哪里？**

---

## ✅ 答案

### 未排程数量存放在这个字段：

```
sales_order_items.pending_qty
```

### 更新逻辑

```
【排程前】
  pending_qty = 100 卷

【用户排程 60 卷】
  schedule_qty = 60

【排程提交后】
  pending_qty = 100 - 60 = 40 卷 ← 自动更新

【后续查询待排程订单】
  只显示 pending_qty > 0 的订单
  现在只显示这个订单的 40 卷
  用户可以继续排程这 40 卷
```

---

## 🗂️ 两个关键表

### 表1：销售订单明细表 (sales_order_items)
```
id | order_id | material_code | order_qty | pending_qty | produced_qty | status
1  | 1        | TAPE-001      | 100       | 40 ← 待排   | 0            | pending
```

### 表2：排程关联表 (schedule_order_item)
```
id | schedule_id | order_item_id | schedule_qty | status
1  | 1           | 1             | 60 ← 本次    | pending
2  | 2           | 1             | 30 ← 后续    | pending
3  | 3           | 1             | 10 ← 最后    | pending
```

---

## 🔄 分批排程示例

```
订单：PO-001 - TAPE-001 - 100卷

【第1次排程】
  用户选择：60卷
  pending_qty: 100 → 40
  
【第2次排程】
  系统显示：还有40卷待排
  用户选择：30卷
  pending_qty: 40 → 10
  
【第3次排程】
  系统显示：还有10卷待排
  用户选择：10卷
  pending_qty: 10 → 0 ✓ 完全排程

历史记录（schedule_order_item）：
  排程1：60卷
  排程2：30卷
  排程3：10卷
  总计：100卷 ✓
```

---

## 💾 核心逻辑

| 字段 | 含义 | 何时变化 |
|------|------|---------|
| `order_qty` | 订单原始数量 | 订单创建时 |
| `pending_qty` | **待排程数量** | 每次排程后递减 |
| `schedule_qty` | 本次排程数量 | 排程时记录 |
| `produced_qty` | 已生产数量 | 报工时递增 |

---

## 🛡️ 关键约束

```sql
-- 1. pending_qty 不能为负
ALTER TABLE sales_order_items 
ADD CONSTRAINT check_pending_qty CHECK (pending_qty >= 0);

-- 2. 排程数量不能超过待排数量
schedule_qty ≤ pending_qty  ← 后端验证

-- 3. 待排 + 已生产 ≤ 订单总量
pending_qty + produced_qty ≤ order_qty
```

## 📝 前端实现

```javascript
// 表格添加可编辑的排程数量列
<el-table-column prop="pending_qty" label="待排数量" width="100" />
<el-table-column prop="schedule_qty" label="本次排程" width="100">
  <template slot-scope="{ row }">
    <el-input-number
      v-model="row.schedule_qty"
      :min="0"
      :max="row.pending_qty"  // ← 验证不超过待排
      @change="validateScheduleQty(row)"
    />
  </template>
</el-table-column>

// 验证方法
validateScheduleQty(row) {
  if (row.schedule_qty > row.pending_qty) {
    this.$message.warning(`最多排${row.pending_qty}卷`)
    row.schedule_qty = row.pending_qty
  }
}

// 排程时收集数据
const details = selectedOrders.map(order => ({
  order_item_id: order.id,
  schedule_qty: order.schedule_qty  // 用户输入的值
}))
```

## 🔗 后端实现

```javascript
// 排程后自动更新 pending_qty
UPDATE sales_order_items
SET pending_qty = pending_qty - #{scheduleQty}
WHERE id = #{orderItemId}

// 事务处理：
// 1. 验证 scheduleQty ≤ pendingQty
// 2. 创建 schedule_order_item 记录
// 3. 更新 sales_order_items.pending_qty
// 4. 创建工序任务
```

## ✨ 核心优势

✅ **清晰** - pending_qty 直观显示还能排多少  
✅ **可追溯** - schedule_qty 记录每次排程历史  
✅ **支持分批** - 可以多次从剩余数量中排程  
✅ **数据一致** - 两个表分工明确  
✅ **易验证** - schedule_qty ≤ pending_qty 简单有效
