## 🔍 调试技巧

### 技巧1：添加日志输出

在 `ProductionScheduleServiceImpl.java` 中添加调试日志：

```java
@Override
@Transactional
public ProductionSchedule autoSchedule(List<Long> orderItemIds, String scheduleDate, String operator) {
    log.info("========== 开始自动排程 ==========");
    log.info("排程日期: {}", scheduleDate);
    
    // 查询待排程订单
    List<Map<String, Object>> pendingItems = orderItemMapper.selectPendingOrderItems(params);
    log.info("待排程订单数: {}", pendingItems.size());
    
    List<Map<String, Object>> needCoatingItems = new ArrayList<>();
    List<Map<String, Object>> canRewindItems = new ArrayList<>();
    
    for (Map<String, Object> item : pendingItems) {
        String orderNo = (String)item.get("order_no");
        int requiredRolls = ((Number)item.get("pending_qty")).intValue();
        
        log.info("检查订单: {}, 需要: {} 卷", orderNo, requiredRolls);
        
        // 查询库存
        List<TapeStock> availableStock = stockMapper.selectAvailableJumboStock(thickness, width);
        log.info("  查询到库存: {} 条", availableStock.size());
        
        TapeStock matchedStock = null;
        for (TapeStock stock : availableStock) {
            log.info("  检查库存ID={}, 可用卷数={}", stock.getId(), stock.getTotalRolls());
            if (stock.getTotalRolls() >= requiredRolls) {
                matchedStock = stock;
                log.info("  ✅ 库存充足，进入复卷");
                break;
            }
        }
        
        if (matchedStock == null) {
            log.info("  ❌ 库存不足，进入涂布");
            needCoatingItems.add(item);
        } else {
            canRewindItems.add(item);
        }
    }
    
    log.info("分类结果: 需要涂布={}, 可直接复卷={}", 
             needCoatingItems.size(), canRewindItems.size());
    log.info("========== 排程完成 ==========");
}
```

**查看日志：**
```bash
# Linux/Mac
tail -f logs/mes.log | grep "开始自动排程" -A 20

# Windows PowerShell
Get-Content logs\mes.log -Tail 50 -Wait
```

```
□ 库存表中有数据吗？
  SELECT COUNT(*) FROM tape_stock;
  
□ 库存的规格与订单匹配吗？
  SELECT DISTINCT color_code, thickness, width FROM tape_stock;

  SELECT * FROM equipment WHERE type = 'COATING';

```
□ 排程记录是否创建？
  SELECT * FROM production_schedule ORDER BY id DESC LIMIT 1;

  SELECT * FROM schedule_coating ORDER BY id DESC LIMIT 1;

## 🎯 常见问题排查

### 问题1：排程后没有任何任务生成

**诊断步骤：**
1. 检查是否有错误日志
2. 检查 `production_schedule` 表是否有记录
3. 检查 `schedule_order_item` 表是否有记录
4. 检查 `schedule_coating` 和 `schedule_rewinding` 表

**可能原因：**
```sql
-- 原因1：没有待排程订单
SELECT COUNT(*) FROM sales_order_items WHERE pending_qty > 0;

-- 原因2：涂布/复卷设备不存在
SELECT * FROM equipment WHERE type IN ('COATING', 'REWINDING');

-- 原因3：排程权限问题
SELECT * FROM production_schedule ORDER BY id DESC LIMIT 1;
```

---

### 问题2：订单进入涂布，但pending_qty没有更新

**诊断步骤：**
```sql
-- 查看排程前后的pending_qty
SELECT id, order_no, pending_qty FROM sales_order_items;

-- 查看排程记录中的schedule_qty
SELECT order_item_id, schedule_qty FROM schedule_order_item;

-- 计算是否应该更新
-- pending_qty_after = pending_qty_before - schedule_qty
```

**可能原因：**
- 后端没有执行更新语句
- 事务回滚了
- 没有提交更改

---

### 问题3：相同颜色+厚度的订单没有合并

**诊断步骤：**
```sql
-- 查看涂布任务中的订单关联
SELECT 
  ct.id as task_id,
  ct.task_no,
  ct.color_code,
  ct.thickness,
  COUNT(soi.id) as 订单数
FROM schedule_coating ct
LEFT JOIN schedule_order_item soi ON ct.schedule_id = soi.schedule_id
GROUP BY ct.id;

-- 查看是否有多个task对应同一个颜色+厚度
```

**可能原因：**
- 分组逻辑有问题
- 颜色代码格式不统一（空格、大小写）

**解决方案：**
```sql
-- 检查颜色代码的格式
SELECT DISTINCT color_code, LENGTH(color_code), HEX(color_code)
FROM sales_order_items;
```

### 修改2：改变设备分配策略

**文件：** `ProductionScheduleServiceImpl.java`
**位置：** 第425-428行

```java
// 修改前：轮询分配
Equipment equipment = coatingEquipments.get(equipmentIndex % coatingEquipments.size());

// 修改后：按颜色专属设备分配
Equipment equipment = getEquipmentByColor(colorCode);

// 新增方法
private Equipment getEquipmentByColor(String colorCode) {
    Map<String, Long> colorEquipmentMap = new HashMap<>();
    colorEquipmentMap.put("30", 1L);  // 蓝色用设备1
    colorEquipmentMap.put("20", 2L);  // 绿色用设备2
    colorEquipmentMap.put("50", 3L);  // 红色用设备3
    
    Long equipmentId = colorEquipmentMap.getOrDefault(colorCode, 1L);
    return equipmentMapper.selectById(equipmentId);
}
```

## ✅ 验证清单

排程完成后，执行以下检查：

```
排程验证清单：
├─ □ 排程单号是否生成（production_schedule.schedule_no）
├─ □ 订单明细是否关联（schedule_order_item 记录数正确）
├─ □ 订单分类是否正确
│  ├─ □ 库存充足 → schedule_rewinding
│  └─ □ 库存不足 → schedule_coating
├─ □ pending_qty 是否更新
├─ □ 涂布参数是否设置
│  ├─ □ 颜色、厚度、面积
│  ├─ □ 涂布速度、烘箱温度
│  └─ □ 预计时长
├─ □ 设备是否分配
├─ □ 订单是否已进入工序（coating/rewinding status = pending）
└─ □ 是否可以在前端看到新的工序任务
```
