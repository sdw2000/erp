# 前端分页数据类型修复总结

## 问题描述

前端 `schedule.vue` 中的 el-pagination 组件报错：
```
Expected Number with value 14, got String with value "14"
```

**原因**：后端返回的分页数据 `total` 字段是字符串类型，但 el-pagination 期望 Number 类型。

## 防御性编程建议

为了避免后端数据类型不一致导致的问题，建议采用以下防御措施：

### 在 API 响应拦截器中统一处理（推荐）

修改 `src/api/index.js` 或 `src/utils/request.js`：

```javascript
// response 拦截器
instance.interceptors.response.use(
  response => {
    const data = response.data.data;
    
    // 如果是分页响应，确保 total 为数字
    if (data && typeof data === 'object' && 'total' in data) {
      data.total = Number(data.total);
      data.pageNum = Number(data.pageNum);
      data.pageSize = Number(data.pageSize);
      data.pages = Number(data.pages);
    }
    
    return response;
  },
  error => Promise.reject(error)
);
```

### 也可以在各个业务方法中处理

方式1 - 在方法中转换（本次采用）：
```javascript
this.total = Number(res.data.total) || 0;
```

方式2 - 使用辅助函数：
```javascript
// utils/helper.js
export function normalizePaginationData(data) {
  return {
    ...data,
    total: Number(data.total),
    pageNum: Number(data.pageNum),
    pageSize: Number(data.pageSize),
    pages: Number(data.pages)
  };
}

// 使用
const normalized = normalizePaginationData(res.data);
this.pendingTotal = normalized.total;
```

## 相关问题

### 为什么后端返回字符串？

**可能的原因**：
1. 数据库查询结果的某些字段被转换为字符串
2. JSON 序列化过程中的类型转换
3. 拦截器或过滤器的处理

**解决建议**（后端）：
```java
// 在 Controller 返回前确保数据类型
Map<String, Object> result = new HashMap<>();
result.put("total", pageInfo.getTotal());  // long 类型
result.put("pageNum", pageInfo.getCurrent());  // int 类型
result.put("pageSize", pageInfo.getSize());  // int 类型
result.put("pages", pageInfo.getPages());  // long 类型
```
