# 初始化导入模板设计

本文档统一整理采购、供应商、研发原材料的初始化模板，便于直接填 Excel 后导入。

## 1. 供应商初始化模板

### 建议字段
| 字段 | 说明 | 必填 |
|---|---|---|
| 供应商编码 | 唯一编码，建议手工维护 | 是 |
| 供应商名称 | 完整名称 | 是 |
| 简称 | 常用简称 | 否 |
| 联系人 | 默认联系人 | 否 |
| 联系电话 | 默认电话 | 否 |
| 纳税人识别号 | 发票信息 | 否 |
| 开户行 | 银行信息 | 否 |
| 银行账号 | 银行信息 | 否 |
| 地址 | 通讯地址 | 否 |
| 状态 | 启用 / 停用 | 是 |
| 备注 | 说明信息 | 否 |

### 建议规则
- `供应商编码` 作为导入主键，避免重复建档。
- 如果需要多联系人，建议另建第二张表：`供应商联系人`。

## 2. 采购报价 / 单价初始化模板

当前已调整为双 Sheet 模板，适合做单价初始化。

### Sheet1：报价单
| 字段 | 说明 | 必填 |
|---|---|---|
| 报价单号 | 同一单据的唯一编号 | 否 |
| 供应商 | 供应商名称 | 是 |
| 联系人 | 默认联系人 | 否 |
| 联系电话 | 联系电话 | 否 |
| 报价日期 | 格式建议 `yyyy-MM-dd` | 是 |
| 有效期至 | 格式建议 `yyyy-MM-dd` | 否 |
| 状态 | `draft/submitted/accepted/rejected/expired` | 否 |
| 备注 | 备注 | 否 |

### Sheet2：报价明细
| 字段 | 说明 | 必填 |
|---|---|---|
| 报价单号 | 关联 Sheet1 | 建议必填 |
| 物料类型 | `film` / `raw` | 是 |
| 物料编码 | 物料编码 | 是 |
| 物料名称 | 物料名称 | 是 |
| 规格 | 如 `100μm*1020mm*5000m` 或 `180kg/桶` | 否 |
| 数量/卷数 | 薄膜填卷数，胶水填桶数 | 是 |
| 单位单价 | 薄膜建议元/㎡，胶水建议元/kg | 是 |
| 备注 | 说明 | 否 |

### 导入规则
- 同一个 `报价单号` 的明细会自动归到同一张报价单。
- 如果不填 `报价单号`，系统会按行创建临时单据，但不建议这样用。
- 薄膜建议统一用 `film`，胶水/原料建议统一用 `raw`。

## 3. 研发原材料初始化模板

研发原材料已统一为一张主表维护，推荐字段如下。

| 字段 | 说明 | 必填 |
|---|---|---|
| 物料编码 | 唯一编码 | 是 |
| 物料名称 | 原材料名称 | 是 |
| 类别 | `film` / `chemical` | 是 |
| 单位 | 如 `卷`、`kg`、`桶`、`m` | 是 |
| 规格 | 规格说明 | 否 |
| 厚度 | 薄膜类可填 | 否 |
| 宽度 | 薄膜类可填 | 否 |
| 长度 | 薄膜类可填 | 否 |
| 性能参数 | JSON 文本或关键参数摘要 | 否 |
| 状态 | 启用 / 停用 | 是 |
| 备注 | 说明 | 否 |

### 性能参数建议
- 薄膜类可维护：厚度公差、拉伸强度、延伸率、表面处理值等。
- 化工类可维护：固含量、粘度、剥离力、开放时间等。
- `性能参数` 建议用 JSON 字符串保存，每个参数按 `standardValue` / `min` / `max` / `unit` / `judgeMode` 维护，例如：`{"widthInspection":{"standardValue":"1201.5","min":"1200","max":"1203","unit":"mm","judgeMode":"range"}}`。
- `judgeMode` 建议取值：`range`(区间)、`min`(≥下限)、`max`(≤上限)、`value`(标准值)。
- 如果参数较多，建议继续放在 `性能参数` 的 JSON 字段里，后续便于做品质判定和自动校验。

## 4. 导入建议

- 先导入供应商，再导入原材料，最后导入单价。
- 所有模板都建议先保留一行示例数据，防止 Excel 表头被改乱。
- 物料编码、供应商编码尽量固定不改，后续数据维护会更稳定。
- 单价初始化最好按月/季度拆分，方便追溯历史价格。

## 5. 当前页面支持情况

- 采购报价页：已支持下载模板、导入、导出，且模板改为双 Sheet 结构。
- 研发原材料页：已支持类别和性能参数维护，并由后端接口生成模板/解析导入。
- 供应商页：已切换为后端模板下载、导入和导出，字段口径与本说明一致。

## 6. 对应接口

- 供应商模板下载：`/purchase/suppliers/template`
- 供应商导入：`/purchase/suppliers/import`
- 供应商导出：`/purchase/suppliers/export`
- 研发原材料模板下载：`/api/tape-formula/raw-material/template`
- 研发原材料导入：`/api/tape-formula/raw-material/import`
- 研发原材料导出：`/api/tape-formula/raw-material/export`
- 采购报价导入模板：前端双 Sheet 模板

## 7. QC 自动判定（来料检验）

- 来料检验页可以直接选择原材料料号，并自动加载该料号维护的 QC 规则。
- 实测值建议填成 JSON，例如：`{"widthInspection":1202,"thicknessTensile":0.5,"coronaBothSides":36}`。
- 系统会根据原材料上的 `standardValue / min / max / unit / judgeMode` 自动判定合格、不合格或待判定。
- `judgeMode` 说明：
	- `range`：区间判定
	- `min`：只看下限，实际值需 `>= min`
	- `max`：只看上限，实际值需 `<= max`
	- `value`：标准值判定，实际值需等于标准值
