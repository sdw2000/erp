# 发货通知状态流转说明

## 📦 发货通知状态

发货通知系统支持以下状态：

| 状态 | 中文名称 | 颜色标识 | 说明 |
|------|---------|---------|------|
| `pending` | 待发货 | 🟡 橙色 | 发货通知已创建，等待确认发货 |
| `shipped` | 已发货 | 🟢 绿色 | 已确认发货，不可再修改 |
| `draft` | 草稿 | ⚪ 灰色 | 草稿状态（预留） |
| `cancelled` | 已作废 | 🔴 红色 | 已取消的发货单（预留） |

## 🔄 状态流转规则

### 1. 创建发货通知
- **默认状态**: `pending` (待发货)
- **操作权限**: admin, sales 角色
- **可执行操作**:
  - ✅ 编辑发货通知
  - ✅ 删除发货通知
  - ✅ 打印发货单
  - ✅ 确认发货

### 2. 确认发货
- **状态变更**: `pending` → `shipped`
- **触发方式**: 点击"确认发货"按钮
- **确认提示**: "确认发货后将无法再修改，是否确认发货？"
- **后端接口**: `POST /delivery/confirm/{id}`

### 3. 已发货状态
- **状态**: `shipped` (已发货)
- **可执行操作**:
  - ✅ 查看详情
  - ✅ 打印发货单
  - ❌ 编辑（已禁用）
  - ❌ 删除（已禁用）
  - ❌ 再次确认（已禁用）

## 🎯 操作指南

### 如何创建发货通知
1. 点击"新增发货通知"按钮
2. 输入销售订单号，失去焦点后自动加载订单信息
3. 系统自动填充：
   - 客户名称
   - 交货地址
   - 联系人和电话
   - 客户订单号
   - 订单明细（可编辑发货数量）
4. 补充发货信息：
   - 发货日期
   - 承运公司
   - 联系电话
   - 备注信息
5. 编辑发货明细：
   - 修改发货数量（卷数）
   - 填写平方米、箱数、毛重等
   - 添加明细备注
6. 点击"保存"创建发货通知
7. **状态自动设置为"待发货"**

### 如何确认发货
1. 在发货通知列表中找到待发货的记录（🟡 橙色标签）
2. 点击"确认发货"按钮
3. 系统弹出确认对话框：
   ```
   确认发货后将无法再修改，是否确认发货？
   ```
4. 点击"确认发货"完成操作
5. 状态更新为"已发货"（🟢 绿色标签）
6. 该发货单的编辑和删除按钮自动隐藏

### 按钮显示逻辑
```
待发货状态 (pending):
  - [详情] [打印] [编辑] [确认发货] [删除]

已发货状态 (shipped):
  - [详情] [打印]
```

## 🎨 使用场景

### 场景1: 正常发货流程
```
1. 销售人员创建发货通知 → pending
2. 仓库打印发货单准备货物
3. 货物出库后点击"确认发货" → shipped
4. 状态锁定，无法修改
```

### 场景2: 需要修改
```
1. 发货通知创建后 → pending
2. 发现数量有误，点击"编辑"修改
3. 修改完成后再次确认发货 → shipped
```

### 场景3: 批量打印
```
1. 筛选出所有待发货订单
2. 逐个打印发货单
3. 打印完成后确认发货
4. 状态更新为已发货
```

## ⚠️ 注意事项

1. **已发货状态不可逆**: 一旦确认发货，状态无法改回待发货
2. **备注功能**: 每个发货明细都可以添加备注，用于保存特殊信息
3. **权限控制**: 仅 admin 和 sales 角色可以操作发货通知
4. **未发货数量**: 创建时自动计算订单数量 - 已发货数量
5. **打印功能**: 所有状态都可以打印发货单
