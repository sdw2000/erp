# ✅ 前端排程表格优化 - 完成总结

## 📋 优化内容

### 1. 本次排程数列 - 改进显示方式

**改进前**：使用 `el-input-number`（带加减号）
```vue
<el-input-number
  v-model="row.schedule_qty"
  :min="0"
  :max="row.pending_qty"
  :step="1"
  size="small"
  style="width: 100%"
/>
```
❌ 问题：加减号遮挡数据，数据看不清

**改进后**：使用纯输入框 `el-input`
```vue
<el-input
  v-model.number="row.schedule_qty"
  type="number"
  size="small"
  placeholder="0"
  style="width: 90px; text-align: center; font-weight: bold; font-size: 14px"
  @input="validateScheduleQty(row)"
  @blur="validateScheduleQty(row)"
/>
```
✅ 优点：
- 纯数字输入框，清晰显示数据
- 加粗字体，更易识别
- 支持 @blur 事件，聚焦失去时验证
- 宽度优化（90px），充足显示空间

### 2. 待排数量列 - 颜色标记

**改进前**：纯数字显示
```vue
<span style="color: #e6a23c; font-weight: bold">
  {{ row.pending_qty }}
</span>
```

**改进后**：使用 el-tag 标签
```vue
<el-tag
  :type="row.pending_qty > 0 ? 'warning' : 'success'"
  size="medium"

  {{ row.pending_qty }}
</el-tag>
```
✅ 优点：
- 待排 > 0：显示**警告色**（橙黄）
- 待排 = 0：显示**成功色**（绿色）
- 更直观的视觉反馈
- 一眼就能看出订单状态

### 3. 验证方法 - 强化逻辑

**改进前**：基础验证
```javascript
validateScheduleQty(row) {
  if (row.schedule_qty > row.pending_qty) {
    row.schedule_qty = row.pending_qty
    this.$message.warning(`排程数量不能超过待排数量 ${row.pending_qty}`)
  }
  if (row.schedule_qty < 0) {
    row.schedule_qty = 0
  }
}
```

**改进后**：完整验证
```javascript
validateScheduleQty(row) {
  // 处理空值和非数字
  if (!row.schedule_qty || row.schedule_qty === '') {
    row.schedule_qty = 0
    return
  }
  
  // 转换为数字
  row.schedule_qty = Number(row.schedule_qty)
  
  // 不能为负数
  if (row.schedule_qty < 0) {
    row.schedule_qty = 0
    this.$message.warning('排程数量不能为负数')
  }
  
  // 不能超过待排数量
  if (row.schedule_qty > row.pending_qty) {
    row.schedule_qty = row.pending_qty
    this.$message.warning(`排程数量不能超过待排数量 ${row.pending_qty}`)
  }
  
  // 整数处理
  row.schedule_qty = Math.floor(row.schedule_qty)
}
```
✅ 优点：
- 处理空值和非数字输入
- 强制整数处理（不允许小数）
- 完整的错误提示
- 更健壮的验证逻辑

## 🔄 业务流程

### 用户操作步骤
1. **打开待排程订单 tab**
   - 系统自动加载待排程订单列表
   - 显示 `pending_qty > 0` 的订单

2. **选择要排程的订单**
   - 勾选左侧 checkbox
   - 可单选或多选

3. **输入本次排程数量**
   - 在"本次排程数"列输入数字
   - 系统实时验证：
     - 不超过待排数量
     - 不为负数
     - 必须是整数

4. **点击批量排程按钮**
   - 系统显示确认对话框
   - 显示：订单数量 + 总排程数

5. **确认排程**
   - 后端处理：
     - 创建排程记录
     - 更新 pending_qty
     - 创建工序任务
   - 前端自动刷新两个列表
