# 📊 未排程数量数据流图

## 1️⃣ 时间线流程

```
【订单创建】
    ↓
    pending_qty = 100 卷
    schedule_qty = NULL (还未排程)
    ↓
【用户查看待排程列表】
    ↓
    显示：pending_qty = 100 卷
    ↓
【用户修改排程数量】
    ↓
    输入：schedule_qty = 60 卷 (≤ pending_qty)
    ↓
【点击【批量排程】】
    ↓
    后端处理：
    ┌─────────────────────────────────┐
    │ 1. 验证：schedule_qty≤pending_qty│
    │ 2. 创建排程记录                 │
    │ 3. 更新：pending_qty -= 60      │
    │ 4. 创建工序任务                 │
    │ 5. 返回成功                     │
    └─────────────────────────────────┘
    ↓
【重新加载待排程列表】
    ↓
    更新：pending_qty = 40 卷 (100-60)
    ↓
【用户继续排程】
    ↓
    输入：schedule_qty = 30 卷 (≤ 40)
    ↓
    → pending_qty = 10 卷 (40-30)
    ↓
【最后一次排程】
    ↓
    输入：schedule_qty = 10 卷 (≤ 10)
    ↓
    → pending_qty = 0 卷 ✓ 完全排程
```

## 4️⃣ 三个关键的数据点

```
【关键数据点1】pending_qty
┌──────────────────────────────┐
│ 含义：还有多少未排程         │
│ 初始值：= order_qty         │
│ 更新时机：每次排程后         │
│ 更新逻辑：-= schedule_qty    │
│ 完成条件：= 0 ✓              │
└──────────────────────────────┘

【关键数据点2】schedule_qty
┌──────────────────────────────┐
│ 含义：本次排程的数量         │
│ 默认值：= pending_qty        │
│ 可修改：Yes (用户输入)       │
│ 约束条件：≤ pending_qty      │
│ 永久记录：在 schedule_order  │
└──────────────────────────────┘

【关键数据点3】produced_qty
┌──────────────────────────────┐
│ 含义：已生产的数量           │
│ 初始值：0                    │
│ 更新时机：报工时             │
│ 更新逻辑：+= output_qty      │
│ 完成条件：= order_qty ✓      │
└──────────────────────────────┘
```

## 5️⃣ 查询语句示例

```sql
-- 查询待排程订单（显示还能排多少）
SELECT 
  order_no, material_code, order_qty,
  pending_qty AS '还能排',
  order_qty - pending_qty AS '已排',
  produced_qty AS '已生'
FROM sales_order_items soi
JOIN sales_orders so ON soi.order_id = so.id
WHERE soi.pending_qty > 0
ORDER BY soi.delivery_date ASC;

-- 查询排程历史（知道排了几次）
SELECT 
  s.schedule_no,
  soi.schedule_qty,
  s.created_at,
  s.status
FROM schedule_order_item soi
JOIN production_schedule s ON soi.schedule_id = s.id
WHERE soi.order_item_id = 1
ORDER BY s.created_at DESC;

-- 验证数据一致性（pending + produced 不超过 order）
SELECT 
  order_no,
  order_qty,
  pending_qty,
  produced_qty,
  (pending_qty + produced_qty) AS total,
  CASE 
    WHEN (pending_qty + produced_qty) > order_qty THEN '❌ ERROR'
    WHEN (pending_qty + produced_qty) = order_qty THEN '✓ OK'
    ELSE '⚠ 未完全排程或生产'
  END AS status
FROM sales_order_items soi
JOIN sales_orders so ON soi.order_id = so.id;
```

## ✅ 核心要点总结

```
┌─────────────────────────────────────────────────────┐
│ 【未排程数量】存放在哪里？                         │
│ → sales_order_items.pending_qty                     │
│                                                     │
│ 【排程历史】存放在哪里？                           │
│ → schedule_order_item.schedule_qty                  │
│                                                     │
│ 【排程前】pending_qty = order_qty                  │
│ 【每次排程后】pending_qty -= schedule_qty           │
│ 【完全排程】pending_qty = 0                         │
│                                                     │
│ 【支持分批排程】可以多次从 pending_qty 中选择     │
└─────────────────────────────────────────────────────┘
```
