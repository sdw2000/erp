# 🔧 涂布排程诊断指南 - 快速排查

## 问题：所有订单都进入了待涂布状态

---

## 🚀 快速诊断（按顺序执行）

### 第1步：检查库存表是否有数据

```sql
-- 在数据库执行
SELECT COUNT(*) as 库存总数 FROM tape_stock;
```

**结果解读：**
- `0` → ❌ **库存表为空**，这就是问题所在！
- `> 0` → ✅ 有库存，继续下一步

---

### 第2步：查看库存详情

```sql
SELECT 
  id,
  color_code,
  thickness,
  width,
  total_rolls,
  status
FROM tape_stock
LIMIT 10;
```

**检查要点：**
- [ ] `color_code` - 颜色代码是否有值？（不能为NULL）
- [ ] `thickness` - 厚度是否有值？（不能为NULL）
- [ ] `width` - 宽度是否有值？（不能为NULL）
- [ ] `total_rolls` - 数量是否 > 0？
- [ ] `status` - 状态是否为 `available`？

**如果看到这样的结果就是问题：**
```
| id | color_code | thickness | width | total_rolls | status    |
|----|----------|-----------|-------|-----------|-----------|
| 1  | NULL     | 100       | 1200  | 0         | available |
| 2  | 30       | NULL      | 1200  | 50        | available |
| 3  | 蓝色     | 100       | NULL  | 100       | canceled  |
```

---

### 第3步：检查订单规格

```sql
-- 查看待排程的订单
SELECT 
  id,
  order_no,
  material_code,
  order_qty,
  pending_qty,
  color_code,
  thickness,
  width
FROM sales_order_items
WHERE pending_qty > 0
LIMIT 10;
```

**检查要点：**
- [ ] 订单的 `color_code` 是否在库存表中存在？
- [ ] 订单的 `thickness` 是否在库存表中存在？
- [ ] 订单的 `width` 是否小于等于库存的 `width`？
- [ ] 订单的 `pending_qty` 是否小于等于库存的 `total_rolls`？

---

### 第4步：模拟库存查询

```sql
-- 模拟系统在排程时查询的SQL
-- 假设订单要求：蓝色(30) + 100μm厚 + 1200mm宽 + 100卷

SELECT * FROM tape_stock
WHERE color_code = '30'      -- 匹配颜色
  AND thickness = 100         -- 匹配厚度
  AND width >= 1200           -- 宽度足够
  AND total_rolls >= 100      -- 库存足够
  AND status = 'available';   -- 状态可用
```

**结果解读：**
- 有结果 → ✅ 库存充足，订单应该进入复卷
- 无结果 → ❌ 库存不足，订单进入涂布

## 🎯 问题排查表

### 情况1：库存表完全为空

**症状：**
```
所有订单 → 全部进入待涂布
```

**原因：**
- 库存数据未导入

**解决方案：**
```sql
-- 1. 导入库存数据
INSERT INTO tape_stock (color_code, thickness, width, total_rolls, status)
VALUES 
  ('30', 100, 1200, 500, 'available'),
  ('20', 150, 1000, 300, 'available'),
  ('50', 120, 1200, 200, 'available');

-- 2. 验证
SELECT * FROM tape_stock;
```

---

### 情况2：库存规格不匹配

**症状：**
```
库存有数据，但订单仍进入待涂布
```

**原因：**
```
库存：蓝色(30) + 100μm + 1200mm + 100卷
订单：蓝色(30) + 150μm + 1200mm + 100卷  ← 厚度不同！

查询时找不到"100μm厚"的库存
→ 订单进入涂布
```

**解决方案：**
```sql
-- 添加匹配的库存规格
INSERT INTO tape_stock (color_code, thickness, width, total_rolls, status)
VALUES ('30', 150, 1200, 100, 'available');  -- ← 添加150μm的库存
```

---

### 情况3：库存数量不足

**症状：**
```
库存：蓝色(30) + 100μm + 1200mm + 50卷
订单：蓝色(30) + 100μm + 1200mm + 100卷  ← 需要100卷

查询时：
WHERE total_rolls >= 100  ← 库存只有50卷，不满足
→ 订单进入涂布
```

**解决方案：**
```sql
-- 增加库存数量
UPDATE tape_stock 
SET total_rolls = 200
WHERE color_code = '30' AND thickness = 100;
```

---

### 情况4：库存状态不对

**症状：**
```
库存存在，但状态为 'canceled' 或 'reserved'
```

**原因：**
```sql
SELECT * FROM tape_stock
WHERE color_code = '30' 
  AND thickness = 100
  AND status = 'canceled';  ← ❌ 状态不对
```

**解决方案：**
```sql
-- 修改库存状态为可用
UPDATE tape_stock 
SET status = 'available'
WHERE color_code = '30' AND thickness = 100;
```

## 📊 完整诊断脚本

将这个脚本复制到数据库执行，一次性诊断：

```sql
-- ========== 库存诊断 ==========

-- 1. 库存总数
SELECT COUNT(*) as 库存总数 FROM tape_stock;

-- 2. 库存状态分布
SELECT status, COUNT(*) as 数量 FROM tape_stock GROUP BY status;

-- 3. 各规格库存
SELECT color_code, thickness, width, SUM(total_rolls) as 总卷数
FROM tape_stock
GROUP BY color_code, thickness, width
ORDER BY color_code, thickness;

-- ========== 订单诊断 ==========

-- 4. 待排程订单
SELECT COUNT(*) as 待排程订单数 
FROM sales_order_items
WHERE pending_qty > 0;

-- 5. 待排程订单的规格
SELECT DISTINCT color_code, thickness, width
FROM sales_order_items
WHERE pending_qty > 0
ORDER BY color_code, thickness;

-- ========== 匹配度检查 ==========

-- 6. 查看每个待排程订单是否有匹配的库存
SELECT DISTINCT 
  o.id as 订单ID,
  o.order_no as 订单号,
  o.color_code as 订单颜色,
  o.thickness as 订单厚度,
  o.width as 订单宽度,
  o.pending_qty as 待排数量,
  COUNT(s.id) as 库存数量,
  SUM(s.total_rolls) as 可用卷数
FROM sales_order_items o
LEFT JOIN tape_stock s 
  ON o.color_code = s.color_code 
  AND o.thickness = s.thickness 
  AND o.width <= s.width
  AND s.status = 'available'
WHERE o.pending_qty > 0
GROUP BY o.id, o.order_no, o.color_code, o.thickness, o.width, o.pending_qty;

-- ========== 涂布任务诊断 ==========

-- 7. 涂布任务统计
SELECT status, COUNT(*) as 任务数
FROM schedule_coating
GROUP BY status;

-- 8. 各颜色+厚度的涂布任务
SELECT color_code, thickness, COUNT(*) as 任务数, SUM(plan_sqm) as 计划面积
FROM schedule_coating
GROUP BY color_code, thickness;
```

## 📞 常见问题快速答案

**Q: 为什么订单优先进入涂布而不是复卷？**
A: 因为没有匹配的库存。系统会先检查库存，如果有足够的库存就直接复卷，否则进入涂布。

**Q: 如何强制订单进入复卷而不是涂布？**
A: 确保库存表中有足够的库存，并且规格完全匹配。

**Q: 如何改变分组策略（比如不按厚度分组）？**
A: 修改 `ProductionScheduleServiceImpl.java` 第227-232行的分组逻辑。

**Q: 涂布后会生成什么任务？**
A: 涂布完成后，系统会自动生成复卷、分切、分条等后续工序任务。

**Q: 如何查看排程后的涂布任务？**
A: 前端进入"计划管理" → "涂布计划"标签，查看待涂布的任务。
