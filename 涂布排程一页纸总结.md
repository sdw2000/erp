# 🎯 涂布排程逻辑 - 一页纸总结

## 您的问题
**"为什么排程时所有订单都进入了待涂布状态？"**

---

## 答案
✅ **这是正常的！** 因为系统检查了库存，库存不足或不存在，所以订单进入涂布流程。

---

## 核心逻辑（3句话）

1. **排程时自动检查库存**
   ```
   有库存 → 直接复卷
   无库存 → 进入涂布
   ```

2. **库存检查条件**
   ```
   WHERE color_code = 订单.color
     AND thickness = 订单.厚度
     AND width >= 订单.宽度
     AND total_rolls >= 订单.数量
     AND status = 'available'
   ```

3. **如果没有匹配的库存，订单就进入涂布**
   ```
   这很正常，因为需要先涂布来补充库存
   ```

---

## 决策流程图

```
订单（需要100卷蓝色膜）
  ↓
检查库存【SELECT...WHERE color='30' AND thickness=100...】
  ├─ ✅ 有库存（500卷）→ 直接复卷
  └─ ❌ 无库存 → 进入涂布 ← 您的情况

按颜色+厚度分组
  ↓
为每组创建涂布任务
  ↓
订单进入"待涂布"
```

---

## 如何解决？

### 方案1：添加库存（5分钟）

```sql
-- 查看现在的库存
SELECT COUNT(*) FROM tape_stock;

-- 如果为0，添加库存
INSERT INTO tape_stock (color_code, thickness, width, total_rolls, status)
VALUES 
  ('30', 100, 1200, 1000, 'available'),
  ('20', 150, 1000, 800, 'available');

-- 重新排程，订单会进入复卷
```

### 方案2：理解这是正常流程

有时涂布是必需的：
- ✅ 库存为空 → 必须涂布来创建初始库存
- ✅ 库存不足 → 涂布补充库存
- ✅ 这个设计是对的！优先复卷（利用库存），其次涂布（创建库存）

## 关键SQL

### 查询库存是否足够

```sql
-- 订单需要：蓝色(30) + 100μm厚 + 100卷
SELECT * FROM tape_stock
WHERE color_code = '30'
  AND thickness = 100
  AND width >= 1200
  AND total_rolls >= 100
  AND status = 'available';

-- 如果有结果 → 订单进入复卷 ✅
-- 如果无结果 → 订单进入涂布 ⚠️
```

### 查询所有需要涂布的订单

```sql
-- 当前所有正在涂布的订单
SELECT 
  soi.order_no,
  soi.pending_qty,
  soi.color_code,
  soi.thickness,
  sc.task_no as 涂布任务号,
  sc.status
FROM sales_order_items soi
LEFT JOIN schedule_coating sc ON soi.order_id = sc.schedule_id
WHERE sc.status = 'pending';
```

## 常见情况

| 情况 | 原因 | 解决方案 |
|------|------|--------|
| 所有订单都进入涂布 | 库存为空或不匹配 | 添加库存 |
| 部分订单进入涂布 | 该规格库存不足 | 增加库存数量 |
| 没有任何订单进入涂布 | 库存充足 | 正常，订单进入复卷 |
| 涂布分组太多 | 分组规则导致 | 修改分组逻辑 |

```java
return colorCode + "_" + thickness;
```

```java

```

### 改为按客户+颜色分组
