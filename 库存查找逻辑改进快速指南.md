# 🔄 库存查找逻辑改进 - 快速对比与实施

## 核心问题与解决方案

### ❌ 当前逻辑的问题

```
现在：按 颜色+厚度 查找库存
问题：
1. 忽略了料号，不够准确
2. 无法区分成品、复卷、母卷
3. 不能充分利用各类库存
4. 涂布数量固定，不科学
```

### ✅ 改进后的逻辑

```
改为：按 料号 为基准，3级查找

第1级 → 成品（完全匹配） → 直接出库
第2级 → 复卷（长度足够） → 分切
第3级 → 母卷（宽度足够） → 复卷+分切
未找到 → 涂布（数量基于销量）
```

---

## 📊 三级库存查找详细说明

### 级别1：成品库存 (Finished Product)

**查找条件：**
```sql
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?                 -- 颜色相同
  AND thickness = ?                  -- 厚度相同
  AND width = ?                      -- 宽度精确匹配
  AND length = ?                     -- 长度精确匹配
  AND total_rolls >= 需要的卷数
  AND product_type = 'finished'      -- 标记为成品
  AND status = 'available'
```

**适用场景：**
- ✅ 订单要求的规格与库存完全一致
- ✅ 可以直接出库，无需任何后续工序

**例子：**
```
订单需求：TAPE-001 | 蓝30 | 100μm | 1200mm | 1000m | 100卷
库存查询：是否有这样的成品库存？
结果：有 → 直接锁定出库！订单完成 ✓
```

---

### 级别2：复好卷库存 (Rewound Product)

**查找条件：**
```sql
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?
  AND thickness = ?
  AND width = ?                      -- 宽度精确匹配
  AND length >= ?                    -- 长度足够即可（不需要精确）
  AND total_rolls >= 需要的卷数
  AND product_type = 'rewound'       -- 标记为复卷后的产品
  AND status = 'available'
```

**适用场景：**
- ✅ 订单宽度与库存宽度相同
- ✅ 但长度可能比库存短（可从长卷分切出来）
- ✅ 直接进入分切工序，无需涂布和复卷

**例子：**
```
订单需求：TAPE-001 | 蓝30 | 100μm | 1200mm | 1000m | 100卷
库存查询：是否有相同料号、颜色、厚度、宽度的复好卷？
库存情况：TAPE-001 | 蓝30 | 100μm | 1200mm | 2000m （长度充足！）
结果：找到 → 订单进入分切流程 ✓
```

---

### 级别3：母卷库存 (Jumbo Roll)

**查找条件：**
```sql
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?
  AND thickness = ?
  AND width >= ?                     -- 宽度足够即可（可大于订单宽度）
  AND total_rolls >= 需要的卷数
  AND product_type = 'jumbo'         -- 标记为母卷（未复卷）
  AND status = 'available'
```

**适用场景：**
- ✅ 订单宽度 ≤ 母卷宽度（可从宽卷裁出）
- ✅ 需要先复卷到订单宽度，再分切到订单长度
- ✅ 需要经过复卷+分切两个工序

**例子：**
```
订单需求：TAPE-001 | 蓝30 | 100μm | 1200mm | 1000m | 100卷
库存查询：是否有相同料号、颜色、厚度的母卷？
库存情况：TAPE-001 | 蓝30 | 100μm | 1500mm | 5000m 母卷
结果：找到 → 订单进入复卷+分切流程 ✓
  步骤1：复卷母卷 1500mm → 1200mm（复卷）
  步骤2：分切复卷 1200mm × 5000m → 1000m × 100卷（分切）
```

## 🎯 涂布数量的科学计算

### 步骤1：统计现有库存

```sql
-- 查询某料号的所有库存（母卷和复卷）的总平方米数
SELECT 
  SUM(width * length * total_rolls / 1000000) as 库存平方米数
FROM tape_stock
WHERE material_code = 'TAPE-001'
  AND product_type IN ('jumbo', 'rewound')
  AND status = 'available'
```

**例子：**
```
TAPE-001 现有库存：
├─ 母卷A：1500mm × 5000m × 10卷 = 75,000,000 mm² = 750万m²
├─ 复卷B：1200mm × 1000m × 50卷 = 6,000,000 mm² = 60万m²
├─ 复卷C：1200mm × 500m × 30卷 = 1,800,000 mm² = 18万m²
└─ 合计：828万m²
```

---

### 步骤2：计算月均销量

```sql
-- 查询最近3个月的销售数据
SELECT 
  DATE_TRUNC('month', delivery_date) as 月份,
  SUM(delivery_qty) as 月销量_平方米
FROM sales_delivery_records
WHERE material_code = 'TAPE-001'
  AND delivery_date >= DATE_SUB(NOW(), INTERVAL 3 MONTH)
GROUP BY DATE_TRUNC('month', delivery_date)
```

**例子：**
```
TAPE-001 最近3个月销量：
├─ 2025年11月：200万m²
├─ 2025年12月：250万m²
├─ 2026年1月：180万m²
└─ 合计平均：(200+250+180)/3 = 210万m²/月
```

---

### 步骤3：计算库存天数

```
库存天数 = 现有库存 / 月均销量 × 30
        = 828万 / 210万 × 30
        = 118天
```

**库存充足度：**
```
118天 ≈ 3.9个月，库存充足！
```

---

### 步骤4：根据库存天数确定涂布数量

```
IF 库存天数 < 30天：
  ┌─ 库存即将耗尽
  └─ 涂布数量 = 月均销量 × 12个月  （备足1年货）

ELSE IF 库存天数 < 60天：
  ┌─ 库存不足
  └─ 涂布数量 = 月均销量 × 6个月   （备足6个月）

ELSE IF 库存天数 < 90天：
  ┌─ 库存一般
  └─ 涂布数量 = 月均销量 × 3个月   （备足3个月）

ELSE：
  ┌─ 库存充足
  └─ 涂布数量 = 0                 （暂不涂布）
```

**例子：**
```
TAPE-001 的情况：
- 库存天数 = 118天 > 90天
- 结论：库存充足
- 涂布决策：暂不涂布，涂布数量 = 0

下个月自动重新评估库存情况...
```

## 📝 数据库调整清单

### 需要修改的表

| 表名 | 修改 | 说明 |
|------|------|------|
| tape_stock | 新增字段 | material_code, product_type |
| sales_delivery_records | 新建表 | 记录交付历史 |

### SQL脚本

```sql
-- 1. 修改tape_stock表
ALTER TABLE tape_stock ADD COLUMN (
  material_code VARCHAR(50) NOT NULL COMMENT '料号',
  product_type ENUM('jumbo', 'rewound', 'finished') 
               COMMENT '产品类型：母卷/复卷/成品',
  original_qty INT DEFAULT 0 COMMENT '原始数量',
  remaining_qty INT DEFAULT 0 COMMENT '剩余数量'
);

-- 2. 添加索引
CREATE INDEX idx_material_lookup 
ON tape_stock(material_code, product_type, status);

-- 3. 创建销售交付记录表
CREATE TABLE sales_delivery_records (
  id INT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(50),
  material_code VARCHAR(50),
  delivery_qty DECIMAL(10, 2) COMMENT '交付平方米数',
  delivery_date DATE,
  actual_qty INT COMMENT '交付卷数',
  customer_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_material_date (material_code, delivery_date)
);

-- 4. 数据迁移：为现有库存补充material_code
-- （需要根据实际情况手动维护）
UPDATE tape_stock SET material_code = '' WHERE material_code IS NULL;
UPDATE tape_stock SET product_type = 'finished' WHERE product_type IS NULL;
```

## 🎯 最终总结

### 关键改进点

1. **★ 以料号为基准**
   - 不再只看颜色+厚度
   - 料号相同才能配对

2. **★ 3级逐级查找**
   - 成品 → 复卷 → 母卷 → 涂布
   - 充分利用各类库存

3. **★ 科学的涂布决策**
   - 基于库存天数
   - 基于月均销量
   - 不再盲目涂布

4. **★ 完整的业务支撑**
   - 直接出库、分切、复卷+分切、涂布
   - 各种场景都覆盖

这样的设计更符合实际生产流程，更科学，成本更低！
