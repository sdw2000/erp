## 测试数据库验证

```sql
-- 查看测试订单
SELECT * FROM sales_orders WHERE order_no LIKE 'SO-202412-%';

-- 查看订单明细
SELECT o.order_no, i.* 
FROM sales_order_items i
JOIN sales_orders o ON i.order_id = o.id
WHERE o.order_no LIKE 'SO-202412-%';
```

## 常见问题排查

### 问题1：点击Tab键后没有反应
- **原因**：订单号输入框的 blur 事件未触发
- **解决**：检查前端代码中是否正确绑定了 `@blur="fetchOrderDetails"`

### 问题2：提示"未找到该销售订单"
- **检查**：
  1. 确认订单号拼写正确（区分大小写）
  2. 检查数据库中是否存在该订单：
     ```sql
     SELECT * FROM sales_orders WHERE order_no = 'SO-202412-001';
     ```

### 问题3：订单信息填充了但没有明细
- **检查**：
  1. 查看浏览器控制台是否有错误
  2. 检查后端接口返回的数据结构
  3. 确认 order.items 数组有数据

### 问题4：中文显示为乱码
- **检查**：
  1. 数据库连接字符集：application.properties 中是否包含 `characterEncoding=UTF-8`
  2. 浏览器F12控制台查看响应数据编码
