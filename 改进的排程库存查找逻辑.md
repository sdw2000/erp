## 核心改进：三级库存查找策略

### 第1级：成品库存查找
**目标：** 找到完全一样的成品

```sql
-- 查找完全一样的成品库存
SELECT * FROM tape_stock
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?                 -- 颜色相同
  AND thickness = ?                  -- 厚度相同
  AND width = ?                      -- 宽度相同（精确匹配）
  AND length = ?                     -- 长度相同（精确匹配）
  AND total_rolls >= ?               -- 数量足够
  AND status = 'available'
LIMIT 1;
```

**结果处理：**
- ✅ **找到** → 直接锁定出库，订单完成！
- ❌ **未找到** → 继续第2级

---

### 第2级：复好卷库存查找
**目标：** 找复好卷的物料，长度相同或足够

```sql
-- 查找复好卷的库存（已经过复卷工序）
SELECT * FROM tape_stock
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?
  AND thickness = ?
  AND width = ?
  AND length >= ?                    -- 长度足够即可（可分切）
  AND product_type = 'rewound'       -- 标记为已复卷
  AND total_rolls >= ?
  AND status = 'available'
LIMIT 1;
```

**结果处理：**
- ✅ **找到** → 订单进入分切流程（不需要涂布和复卷）
- ❌ **未找到** → 继续第3级

---

### 第3级：母卷库存查找
**目标：** 找母卷（涂好的），可以复卷后分切

```sql
-- 查找母卷库存（涂好的，尚未复卷）
SELECT * FROM tape_stock
WHERE material_code = ?              -- ★ 料号必须相同
  AND color_code = ?
  AND thickness = ?
  AND width >= ?                     -- 母卷宽度足够即可
  AND product_type = 'jumbo'         -- 标记为母卷
  AND total_rolls >= ?
  AND status = 'available'
LIMIT 1;
```

**结果处理：**
- ✅ **找到** → 订单进入复卷+分切流程
- ❌ **未找到** → 订单进入涂布流程

## 🎨 涂布数量的计算逻辑

### 第1步：统计现有库存中相同料号的产能

```sql
-- 统计：TAPE-001 现有的所有库存平方数（母卷或成品）
SELECT 
  SUM(width * length * total_rolls / 1000000) as 现有库存平方数
FROM tape_stock
WHERE material_code = 'TAPE-001'
  AND status = 'available'
  AND (product_type IN ('jumbo', 'rewound') OR product_type IS NULL);
```

**例子：**
```
现有库存：
├─ 母卷A：1500mm × 5000m × 10卷 = 75亿mm² = 750万m²
├─ 成品B：1200mm × 1000m × 50卷 = 6亿mm² = 60万m²
├─ 复好卷C：1200mm × 500m × 30卷 = 1.8亿mm² = 18万m²
└─ 总计：828万m²
```

---

### 第2步：获取客户最近3个月销量

```sql
-- 统计：最近3个月 TAPE-001 的销量
SELECT 
  SUM(delivery_qty) as 三月销量,
  SUM(delivery_qty) / 3 as 月均销量
FROM sales_delivery_records
WHERE material_code = 'TAPE-001'
  AND delivery_date >= DATE_SUB(NOW(), INTERVAL 3 MONTH);
```

**例子：**
```
三个月销量：
├─ 1月：200万m²
├─ 2月：250万m²
├─ 3月：180万m²
├─ 小计：630万m²
└─ 月均：210万m²

库存天数 = 现有库存 / 月均销量 * 30
         = 828 / 210 * 30
         = 118天（约3.9个月，库存充足）
```

---

### 第3步：确定涂布数量

```
IF 库存天数 < 30天 THEN
  需要补货涂布
  
  涂布数量 = 月均销量 * 12  -- 备货1年份
  
ELSE IF 库存天数 < 60天 THEN
  需要适量补货
  
  涂布数量 = 月均销量 * 6   -- 备货6个月
  
ELSE IF 库存天数 < 90天 THEN
  可适量少补
  
  涂布数量 = 月均销量 * 3   -- 备货3个月
  
ELSE
  库存充足，不涂布
  涂布数量 = 0
END IF
```

**例子：**
```
库存天数 = 118天 > 90天
→ 库存充足，暂不需要涂布
→ 涂布数量 = 0

下个月重新评估库存情况...
```

## 🔄 改进后的排程流程

```java
@Transactional
public ProductionSchedule autoSchedule(List<Long> orderItemIds, 
                                      String scheduleDate, 
                                      String operator) {
    
    List<Map<String, Object>> pendingItems = queryPendingOrders(orderItemIds);
    
    // 分类结果
    List<Map<String, Object>> directShipItems = new ArrayList<>();      // 成品直接出库
    List<Map<String, Object>> slittingItems = new ArrayList<>();        // 分切
    List<Map<String, Object>> rewindSlittingItems = new ArrayList<>();  // 复卷+分切
    List<Map<String, Object>> coatingItems = new ArrayList<>();         // 涂布
    
    for (Map<String, Object> item : pendingItems) {
        String materialCode = (String)item.get("material_code");
        
        // ★ 第1级：查找成品
        TapeStock finishedProduct = tapeStockMapper
            .selectExactMatch(materialCode, item);
        
        if (finishedProduct != null && finishedProduct.getTotalRolls() >= requiredQty) {
            directShipItems.add(item);
            continue;
        }
        
        // ★ 第2级：查找复好卷
        TapeStock rewoundProduct = tapeStockMapper
            .selectRewindedMatch(materialCode, item);
        
        if (rewoundProduct != null && canSlitToLength(rewoundProduct, requiredLength)) {
            slittingItems.add(item);
            item.put("sourceStock", rewoundProduct);
            continue;
        }
        
        // ★ 第3级：查找母卷
        TapeStock jumboProduct = tapeStockMapper
            .selectJumboMatch(materialCode, item);
        
        if (jumboProduct != null && canRewindAndSlit(jumboProduct, item)) {
            rewindSlittingItems.add(item);
            item.put("sourceStock", jumboProduct);
            continue;
        }
        
        // ★ 没有库存：进入涂布
        coatingItems.add(item);
    }
    
    // 创建排程
    ProductionSchedule schedule = createSchedule(scheduleDate);
    
    // 分别处理
    if (!directShipItems.isEmpty()) {
        generateDirectShipment(schedule, directShipItems);
    }
    
    if (!slittingItems.isEmpty()) {
        generateSlittingTasks(schedule, slittingItems);
    }
    
    if (!rewindSlittingItems.isEmpty()) {
        generateRewindAndSlittingTasks(schedule, rewindSlittingItems);
    }
    
    if (!coatingItems.isEmpty()) {
        // 计算涂布数量
        Map<String, Integer> coatingQuantities = calculateCoatingQty(coatingItems);
        generateCoatingTasks(schedule, coatingItems, coatingQuantities);
    }
    
    return getScheduleById(schedule.getId());
}

// 计算涂布数量
private Map<String, Integer> calculateCoatingQty(List<Map<String, Object>> coatingItems) {
    Map<String, Integer> result = new HashMap<>();
    
    for (Map<String, Object> item : coatingItems) {
        String materialCode = (String)item.get("material_code");
        
        // 查询现有库存平方数
        BigDecimal currentStock = tapeStockMapper
            .sumInventorySquareMeters(materialCode);
        
        // 查询最近3个月销量
        BigDecimal monthlySales = tapeStockMapper
            .getAverageMonthlySales(materialCode, 3);
        
        // 计算库存天数
        int inventoryDays = calculateInventoryDays(currentStock, monthlySales);
        
        // 根据库存天数确定涂布数量
        int coatingQty;
        if (inventoryDays < 30) {
            coatingQty = monthlySales.multiply(new BigDecimal(12)).intValue();
        } else if (inventoryDays < 60) {
            coatingQty = monthlySales.multiply(new BigDecimal(6)).intValue();
        } else if (inventoryDays < 90) {
            coatingQty = monthlySales.multiply(new BigDecimal(3)).intValue();
        } else {
            coatingQty = 0;  // 库存充足，不涂布
        }
        
        result.put(materialCode, coatingQty);
    }
    
    return result;
}
```

## 🛠️ 新增的DAO查询方法

### TapeStockMapper.java

```java
// 精确匹配：成品完全相同
TapeStock selectExactMatch(String materialCode, Map<String, Object> orderItem);

// 模糊匹配：复好卷（长度可充足）
TapeStock selectRewindedMatch(String materialCode, Map<String, Object> orderItem);

// 母卷匹配：宽度足够即可
TapeStock selectJumboMatch(String materialCode, Map<String, Object> orderItem);

// 统计：某料号的库存总平方米
BigDecimal sumInventorySquareMeters(String materialCode);

// 统计：最近N个月的平均销量
BigDecimal getAverageMonthlySales(String materialCode, int months);
```

### 对应的SQL

      AND thickness = #{item.thickness}

      AND total_rolls >= #{item.qty}
      AND status = 'available'
    LIMIT 1

<!-- 统计库存平方数 -->
<select id="sumInventorySquareMeters" resultType="BigDecimal">
    SELECT COALESCE(SUM(width * length * total_rolls / 1000000), 0)
    FROM tape_stock
    WHERE material_code = #{materialCode}
      AND status = 'available'
      AND product_type IN ('jumbo', 'rewound')
</select>

<!-- 平均月销量 -->
<select id="getAverageMonthlySales" resultType="BigDecimal">
    SELECT COALESCE(AVG(monthly_qty), 0)
    FROM (
        SELECT DATE_TRUNC('month', delivery_date) as month,
               SUM(delivery_qty) as monthly_qty
        FROM sales_delivery_records
        WHERE material_code = #{materialCode}
          AND delivery_date >= DATE_SUB(NOW(), INTERVAL #{months} MONTH)
        GROUP BY DATE_TRUNC('month', delivery_date)
    ) subquery
</select>

## 📊 比较：旧逻辑 vs 新逻辑

| 维度 | 旧逻辑 | 新逻辑 |
|------|-------|-------|
| **查找基准** | 颜色+厚度 | ★ 料号（关键） |
| **库存层级** | 无区分 | 3级：成品→复卷→母卷 |
| **成品直接出库** | ❌ 不支持 | ✅ 支持 |
| **分切直接来源** | ❌ 不支持 | ✅ 支持（复好卷） |
| **复卷+分切** | ❌ 不支持 | ✅ 支持（母卷） |
| **涂布决策** | 固定 | ✅ 动态（根据库存天数） |
| **涂布数量** | 等面积 | ✅ 基于月销量 |
| **库存利用** | 低 | ✅ 高 |
