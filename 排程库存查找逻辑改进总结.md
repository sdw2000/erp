# 🎯 排程库存查找逻辑改进 - 完整方案总结

## 您提出的核心改进建议

### 问题1：查找基准错误
```
❌ 当前：按 颜色+厚度 查找
✅ 改进：按 料号 查找（料号对了才能配对）
```

### 问题2：库存层级不清
```
❌ 当前：不区分库存类型
✅ 改进：3级查找
  ├─ 第1级：成品（完全匹配）
  ├─ 第2级：复卷（长度足够）
  └─ 第3级：母卷（宽度足够）
```

### 问题3：涂布逻辑不科学
```
❌ 当前：固定涂布，不考虑库存
✅ 改进：动态涂布
  ├─ 库存充足（>90天）→ 不涂布
  ├─ 库存不足（60-90天）→ 备3个月
  ├─ 库存紧张（30-60天）→ 备6个月
  └─ 库存即将耗尽（<30天）→ 备12个月
```

## 核心改进方案

### 方案概述

```
排程流程重新设计：

【用户排程】
   ↓
【查询待排程订单】
   ↓
【对每个订单进行库存检查】
   ├─ 第1级：料号+成品的完全配对 → 直接出库 ✓
   ├─ 第2级：料号+复卷的长度可配对 → 分切 ✓
   ├─ 第3级：料号+母卷的宽度可配对 → 复卷+分切 ✓
   └─ 都没有 → 涂布（数量基于销量预测）✓
   ↓
【生成对应的工序任务】
```

## 三级库存查找的详细规则

### 级别1：成品库存 (Product Type = 'finished')

**查找SQL：**
```sql
SELECT * FROM tape_stock
WHERE material_code = '订单的料号'
  AND color_code = '订单的颜色'
  AND thickness = '订单的厚度'
  AND width = '订单的宽度'           -- 精确匹配
  AND length = '订单的长度'          -- 精确匹配
  AND total_rolls >= '订单的卷数'
  AND product_type = 'finished'
  AND status = 'available'
```

**处理方式：**
- ✅ 找到 → 直接锁定出库，订单完成！
- ❌ 未找到 → 进入第2级

**优势：**
- 不需要任何后续工序
- 最快交货
- 最低成本

---

### 级别2：复好卷库存 (Product Type = 'rewound')

**查找SQL：**
```sql
SELECT * FROM tape_stock
WHERE material_code = '订单的料号'
  AND color_code = '订单的颜色'
  AND thickness = '订单的厚度'
  AND width = '订单的宽度'           -- 精确匹配
  AND length >= '订单的长度'         -- ★ 足够即可，可长于订单
  AND total_rolls >= '订单的卷数'
  AND product_type = 'rewound'       -- 已经过复卷
  AND status = 'available'
```

**处理方式：**
- ✅ 找到 → 进入分切流程（分切到订单长度）
- ❌ 未找到 → 进入第3级

**优势：**
- 跳过涂布和复卷两个工序
- 周期短（只需分切）
- 成本低

**例子：**
```
订单需求：1200mm × 1000m
库存有：1200mm × 2000m（超过订单长度）
→ 可以直接分切，无需复卷
```

---

### 级别3：母卷库存 (Product Type = 'jumbo')

**查找SQL：**
```sql
SELECT * FROM tape_stock
WHERE material_code = '订单的料号'
  AND color_code = '订单的颜色'
  AND thickness = '订单的厚度'
  AND width >= '订单的宽度'          -- ★ 足够即可，可宽于订单
  AND total_rolls >= '订单的卷数'
  AND product_type = 'jumbo'         -- 未复卷的母卷
  AND status = 'available'
```

**处理方式：**
- ✅ 找到 → 进入复卷+分切流程
  - 先复卷：从母卷的宽度裁到订单宽度
  - 再分切：分切到订单长度
- ❌ 未找到 → 进入涂布流程

**优势：**
- 充分利用现有母卷
- 避免不必要的涂布

**例子：**
```
订单需求：1200mm × 1000m
库存母卷：1500mm × 5000m
处理步骤：
  1. 复卷 1500mm → 1200mm （从母卷裁一张）
  2. 分切 1200mm × 5000m → 1000m × 100卷
结果：订单完成，无需涂布
```

## 涂布数量的动态计算

### 计算步骤

#### 步骤1：统计现有库存
```sql
SELECT SUM(width * length * total_rolls / 1000000) as 总平方米
FROM tape_stock
WHERE material_code = '料号'
  AND product_type IN ('jumbo', 'rewound')
  AND status = 'available'
```

#### 步骤2：查询最近3个月销量
```sql
SELECT SUM(delivery_qty) / 3 as 月均销量
FROM sales_delivery_records
WHERE material_code = '料号'
  AND delivery_date >= DATE_SUB(NOW(), INTERVAL 3 MONTH)
```

#### 步骤3：计算库存天数
```
库存天数 = 现有库存 / 月均销量 × 30
```

#### 步骤4：根据库存天数确定涂布数量
```
IF 库存天数 < 30天：
  涂布数量 = 月均销量 × 12    （备1年）

ELSE IF 库存天数 < 60天：
  涂布数量 = 月均销量 × 6     （备6月）

ELSE IF 库存天数 < 90天：
  涂布数量 = 月均销量 × 3     （备3月）

ELSE：
  涂布数量 = 0               （不涂布，库存充足）
```

### 例子

```
TAPE-001 的情况：

【步骤1】统计库存
  母卷A：1500mm × 5000m × 10卷 = 75亿mm² = 750万m²
  复卷B：1200mm × 1000m × 50卷 = 6亿mm² = 60万m²
  合计：810万m²

【步骤2】查询月销量
  2025年11月：200万m²
  2025年12月：250万m²
  2026年1月：180万m²
  月均：(200+250+180)/3 = 210万m²

【步骤3】计算库存天数
  库存天数 = 810 / 210 × 30 = 115天

【步骤4】确定涂布量
  115天 > 90天
  → 库存充足，涂布数量 = 0
  → 暂不涂布
```

## 数据库调整

### 需要新增的字段

#### tape_stock 表

```sql
ALTER TABLE tape_stock ADD COLUMN (
  material_code VARCHAR(50) NOT NULL,
  product_type ENUM('jumbo', 'rewound', 'finished')
);

CREATE INDEX idx_material_lookup 
ON tape_stock(material_code, product_type, status);
```

#### 新建表：sales_delivery_records

```sql
CREATE TABLE sales_delivery_records (
  id INT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(50),
  material_code VARCHAR(50),
  delivery_qty DECIMAL(10, 2),
  delivery_date DATE,
  actual_qty INT,
  customer_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_material_date (material_code, delivery_date)
);
```
