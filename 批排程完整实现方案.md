# 🎯 批排程数量逻辑 - 完整实现方案

## 核心问题解答

### ❓ 问题
```
PO-001: 订单100卷，用户选择排程60卷
  → 剩余40卷存放在哪里？
  → 如何追踪这40卷？
  → 用户何时再排这40卷？
```

### ✅ 答案（一句话）
```
未排程数量 (pending_qty) 存放在 sales_order_items 表中，
每次排程后自动减少，用户下次查询时可继续排程剩余数量
```

---

## 📊 完整实现方案

### 第1步：前端表格新增列

在**待排程订单**表格中添加**本次排程数**列（可编辑）

```vue
<!-- 原有列 -->
<el-table-column prop="pending_qty" label="待排数量" width="100" />

<!-- 新增列 - 可编辑的排程数量 -->
<el-table-column prop="schedule_qty" label="本次排程" width="120">
  <template slot-scope="{ row }">
    <el-input-number
      v-model="row.schedule_qty"
      :min="0"
      :max="row.pending_qty"
      size="small"
      @change="validateScheduleQty(row)"
    />
  </template>
</el-table-column>
```

### 第2步：初始化和验证

```javascript
// 加载待排程订单
async loadPendingOrders() {
  this.pendingLoading = true
  try {
    const res = await getPendingOrderItems(this.pendingParams)
    // 后端已返回 pending_qty > 0 的订单
    this.pendingOrders = res.data.list.map(order => ({
      ...order,
      schedule_qty: order.pending_qty // 默认 = 待排数量
    }))
    this.pendingTotal = res.data.total
  } catch (error) {
    this.$message.error('加载待排程订单失败')
  } finally {
    this.pendingLoading = false
  }
}

// 验证排程数量
validateScheduleQty(row) {
  // 不能超过待排数量
  if (row.schedule_qty > row.pending_qty) {
    this.$message.warning(`最多只能排${row.pending_qty}卷`)
    row.schedule_qty = row.pending_qty
  }
  // 不能为负数
  if (row.schedule_qty < 0) {
    row.schedule_qty = 0
  }
}
```

### 第3步：批排程时收集数据

```javascript
async batchSchedule() {
  // 验证是否选中了订单
  if (this.selectedOrders.length === 0) {
    this.$message.warning('请先选择要排程的订单')
    return
  }

  // 收集排程数据
  const scheduleDetails = this.selectedOrders.map(order => ({
    order_item_id: order.id,
    schedule_qty: order.schedule_qty // 用户输入的值
  }))

  try {
    this.scheduling = true
    const res = await autoSchedule({
      scheduleDate: this.autoScheduleForm.scheduleDate,
      scheduleType: this.autoScheduleForm.scheduleType,
      details: scheduleDetails // 关键参数
    })

    if (res.code === 0) {
      this.$message.success('排程成功！')
      // 重新加载待排程订单
      // pending_qty 会自动更新为更小的值
      this.loadPendingOrders()
      this.selectedOrders = []
    } else {
      this.$message.error(res.message || '排程失败')
    }
  } catch (error) {
    this.$message.error('排程失败: ' + error.message)
  } finally {
    this.scheduling = false
  }
}
```

### 第4步：后端处理排程（伪代码）

```javascript
// POST /api/production/schedule/auto
function autoSchedule(request) {
  const { scheduleDate, details } = request
  
  // 开始事务
  transaction.begin()
  
  try {
    // 对每个要排程的订单明细进行处理
    for (const detail of details) {
      const { order_item_id, schedule_qty } = detail
      
      // 1. 验证 schedule_qty ≤ pending_qty
      const orderItem = salesOrderItemsDAO.getById(order_item_id)
      if (schedule_qty > orderItem.pending_qty) {
        throw new Error(`排程数量不能超过待排数量${orderItem.pending_qty}`)
      }
      
      // 2. 创建排程记录
      const schedule = productionScheduleDAO.insert({
        schedule_no: generateScheduleNo(),
        schedule_date: scheduleDate,
        status: 'pending'
      })
      
      // 3. 创建排程-订单关联
      scheduleOrderItemDAO.insert({
        schedule_id: schedule.id,
        order_item_id: order_item_id,
        schedule_qty: schedule_qty,  // 记录本次排程数量
        status: 'pending'
      })
      
      // 4. 更新销售订单明细的待排数量 ← 关键步骤
      salesOrderItemsDAO.update(order_item_id, {
        pending_qty: orderItem.pending_qty - schedule_qty,
        updated_at: new Date()
      })
      
      // 5. 创建各工序任务（涂布、复卷等）
      createProcessTasks(schedule.id, order_item_id, schedule_qty)
    }
    
    // 提交事务
    transaction.commit()
    
    return {
      code: 0,
      message: '排程成功',
      data: { scheduleNo: schedule.schedule_no }
    }
  } catch (error) {
    // 回滚事务
    transaction.rollback()
    return { code: 1, message: error.message }
  }
}
```

### 第5步：数据库更新语句

```sql
-- 排程后更新待排数量
UPDATE sales_order_items
SET 
  pending_qty = pending_qty - ?,  -- ? = schedule_qty
  updated_at = NOW()
WHERE id = ?                       -- ? = order_item_id
  AND pending_qty >= ?;            -- 验证不会为负
```

## 🛡️ 数据一致性检查

```sql
-- 定期检查数据一致性
SELECT 
  so.order_no,
  soi.id,
  soi.order_qty,
  soi.pending_qty,
  soi.produced_qty,
  (soi.pending_qty + soi.produced_qty) AS total,
  CASE 
    WHEN (soi.pending_qty + soi.produced_qty) > soi.order_qty 
      THEN '❌ 数据错误：待排+已生 > 订单数'
    WHEN (soi.pending_qty + soi.produced_qty) = soi.order_qty 
      THEN '✓ 数据正常'
    WHEN soi.pending_qty > 0 
      THEN '⚠️ 部分未排程'
    WHEN soi.pending_qty = 0 AND soi.produced_qty < soi.order_qty
      THEN '⚠️ 已排程未完成生产'
    ELSE '✓ 完成'
  END AS status
FROM sales_order_items soi
JOIN sales_orders so ON soi.order_id = so.id
ORDER BY so.order_date DESC;
```

## ✨ 核心优势

| 特性 | 说明 | 好处 |
|------|------|------|
| **双表记录** | pending_qty + schedule_qty | 既能查还能排多少，也能看排程历史 |
| **自动递减** | 每次排程后更新pending_qty | 用户无需手动维护，自动同步 |
| **支持分批** | 可多次排程同一订单 | 灵活应对业务变化 |
| **易于验证** | schedule_qty ≤ pending_qty | 简单有效的约束条件 |
| **数据可追溯** | schedule_order_item记录历史 | 知道排了几次、每次多少 |
| **实时更新** | 前端自动重新加载 | 用户总是看到最新的待排数量 |
