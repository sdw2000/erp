# 🎨 涂布排程逻辑说明 - 完整分析

## 📋 核心问题
**为什么排程后所有订单都进入了"待涂布"状态？**

### 第1步：库存检查逻辑


### 第2步：订单分类规则

```javascript
// 代码位置：ProductionScheduleServiceImpl.java 第189-220行

// 检查库存，区分两类订单：
List<Map<String, Object>> needCoatingItems = new ArrayList<>();      // ← 需要涂布
List<Map<String, Object>> canRewindItems = new ArrayList<>();        // ← 可直接复卷

for (Map<String, Object> item : pendingItems) {
  Integer thickness = item.get("thickness");
  Integer width = item.get("width");
  
  // ★★★ 关键逻辑 ★★★
  // 查询匹配的母卷库存（按厚度匹配，宽度 >= 订单宽度）
  List<TapeStock> availableStock = stockMapper.selectAvailableJumboStock(thickness, width);
  
  int requiredRolls = item.get("pending_qty");
  TapeStock matchedStock = null;
  
  // 检查是否有足够的库存
  for (TapeStock stock : availableStock) {
    if (stock.getTotalRolls() >= requiredRolls) {
      matchedStock = stock;
      break;
    }
  }
  
  if (matchedStock != null) {
    // ✅ 有库存 → 可以直接复卷
    canRewindItems.add(item);
    itemStockMap.put(item, matchedStock);
  } else {
    // ❌ 无库存 → 需要涂布
    needCoatingItems.add(item);
  }
}
```

## 📊 涂布任务生成流程

### 第3步：所有订单进入待涂布的原因

**通常有以下2个原因：**

#### 原因1：库存不足（最常见）
```
订单要求：
  ├─ 颜色：蓝色(30)
  ├─ 厚度：100μm
  ├─ 宽度：1200mm
  ├─ 长度：1000m
  └─ 数量：100卷

系统查询：
  SELECT * FROM tape_stock
  WHERE color_code = '30' 
    AND thickness = 100 
    AND width >= 1200
    AND status = 'available'
    AND total_rolls >= 100
  
结果：没有找到！

处理：
  → 订单被分类为 needCoatingItems（需要涂布）
  → 生成涂布任务
  → 订单进入"待涂布"状态
```

#### 原因2：库存完全为空
```
如果数据库中根本没有任何库存记录，
则所有订单都会被认为"无库存"，
全部进入"需要涂布"流程
```

## 🔄 涂布任务生成逻辑

### 第4步：涂布任务如何创建

```java
// 代码位置：ProductionScheduleServiceImpl.java 第322行
if (!groupedCoatingItems.isEmpty()) {
    generateCoatingTasks(schedule, groupedCoatingItems, operator);
}

// 详细实现：generateCoatingTasks() 第401-473行
private void generateCoatingTasks(ProductionSchedule schedule, 
                                 Map<String, List<Map<String, Object>>> groupedItems,
                                 String operator) {
    
    // 1️⃣ 按颜色+厚度分组
    // groupedItems 的key = "蓝色_100"、"绿色_150" 等
    // groupedItems 的value = 该组的所有订单
    
    // 2️⃣ 为每组创建一个涂布任务
    for (Map.Entry<String, List<Map<String, Object>>> entry : groupedItems.entrySet()) {
        String[] keys = entry.getKey().split("_");
        String colorCode = keys[0];      // 颜色代码
        String thickness = keys[1];      // 厚度
        
        List<Map<String, Object>> items = entry.getValue();  // 该组的所有订单
        
        // 3️⃣ 计算该组的总面积
        BigDecimal totalSqm = BigDecimal.ZERO;
        for (Map<String, Object> item : items) {
            // 面积 = 宽度 × 长度 × 数量
            BigDecimal sqm = width × length × qty;
            totalSqm = totalSqm.add(sqm);
        }
        
        // 4️⃣ 创建涂布任务
        ScheduleCoating coating = new ScheduleCoating();
        coating.setScheduleId(schedule.getId());
        coating.setColorCode(colorCode);
        coating.setThickness(thickness);
        coating.setPlanSqm(totalSqm);
        
        // 分配涂布设备（轮询）
        Equipment equipment = coatingEquipments.get(equipmentIndex % coatingEquipments.size());
        coating.setEquipmentId(equipment.getId());
        
        // 5️⃣ 设置涂布参数
        coating.setCoatingSpeed(20);      // 20米/分钟
        coating.setOvenTemp(120);         // 120℃
        coating.setStatus("pending");     // ← 进入"待涂布"
        
        coatingMapper.insert(coating);
    }
}
```

## 🛡️ 为什么会这样设计？

### ✅ 优点

| 优点 | 说明 |
|------|------|
| **智能库存** | 自动检查库存，有库存直接复卷，无库存才涂布 |
| **成本优化** | 优先使用现有库存，减少涂布成本 |
| **效率提升** | 库存充足的订单跳过涂布，直接进入复卷 |
| **分组优化** | 相同颜色+厚度的订单合并涂布，提高设备利用率 |
| **自动化** | 无需人工判断，完全自动分类处理 |

## 📊 库存查询SQL

### 查询当前库存状态

```sql
-- 查询所有可用库存
SELECT 
  id,
  color_code,
  thickness,
  width,
  total_rolls,
  status
FROM tape_stock
WHERE status = 'available'
ORDER BY color_code, thickness;

-- 查询特定规格的库存
SELECT * FROM tape_stock
WHERE color_code = '30'        -- 蓝色
  AND thickness = 100          -- 100μm
  AND width >= 1200            -- 宽度≥1200mm
  AND total_rolls > 0          -- 有库存
  AND status = 'available';

-- 查询库存不足的规格
SELECT color_code, thickness, COUNT(*) as stock_count
FROM tape_stock
WHERE total_rolls < 100
GROUP BY color_code, thickness;
```

## 💡 总结

**为什么所有订单都进入待涂布？**

👉 **因为库存不足或库存为空！**

系统的设计逻辑是：
```
IF 库存充足 THEN
  订单进入复卷
ELSE
  订单进入涂布
END IF
```

如果所有订单都进入涂布，说明：
1. ❌ 库存表中没有匹配的库存
2. ❌ 库存数量不足
3. ❌ 库存状态不是 `available`

**解决方案：**
- 检查和补充库存数据
- 确保库存规格匹配
- 验证库存状态是否正确
