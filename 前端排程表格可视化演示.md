## 关键列详解

### 待排数量列 (pending_qty)

```
┌───────────────┐
│ 橙黄标签(待排>0) │  ← 显示还有多少未排程
├───────────────┤
│      40       │
│      85       │
│     536       │
└───────────────┘

┌───────────────┐
│ 绿色标签(待排=0) │  ← 表示订单已全部排程
├───────────────┤
│       0       │
└───────────────┘
```

**颜色含义**：
- 🟠 **橙黄色**：还有待排数量，可继续排程
- 🟢 **绿色**：已排程完毕，无需再排

---

### 本次排程数列 (schedule_qty)

```
不带加减号的纯输入框，更清晰地显示数据

┌──────────────┐
│ 输入框(加粗) │  ← 用户在这里输入本次排程数
├──────────────┤
│     60      │  ← 加粗显示，易于识别
│     85      │
│    200      │
│     —       │  ← 若不排程，保持空或0
└──────────────┘
```

**输入特性**：
- ✏️ 可直接输入数字
- 🔒 自动验证：不超过 pending_qty
- 🔢 自动转换为整数（小数会舍去）
- ⚡ 支持 @input 和 @blur 两种验证方式

---

## 输入验证流程图

```
用户输入数字
     ↓
【@input 事件触发】validateScheduleQty()
     ↓
检查是否为空？
  ├─ 是 → 设为 0，返回
  └─ 否 ↓
     ↓
转换为数字
     ↓
检查是否为负？
  ├─ 是 → 显示警告，设为 0
  └─ 否 ↓
     ↓
检查是否超过 pending_qty？
  ├─ 是 → 显示警告，自动改为 pending_qty
  └─ 否 ↓
     ↓
处理小数（Math.floor）
     ↓
✅ 有效的排程数量
```

## 用户操作示例

### 场景1：批量排程

**操作步骤**：
```
1. 打开"待排程订单" tab
   ↓
2. 勾选前两个订单的 checkbox
   ✓ PO-001 (待排40)
   ✓ PO-002 (待排85)
   ↓
3. 在"本次排程数"列输入
   PO-001 → 60 (提示: 最多40，自动改为40)
   PO-002 → 85
   ↓
4. 点击"批量排程(2)"按钮
   ↓
5. 确认对话框出现
   "确认排程 2 个订单，总计 125 卷？"
   ↓
6. 点击"确定"
   ↓
7. ✅ 排程成功提示
   "排程成功！已排程 2 个订单"
   ↓
8. 表格自动刷新
   - PO-001: pending_qty 40→40 (不变，完全排程)
   - PO-002: pending_qty 85→0 (完全排程)
```

### 场景2：分批排程（同一订单排多次）

**订单 PO-001：100卷**

```
【第1次排程】
  pending_qty: 100 → 40 (排程60卷)
  
【用户下次打开待排程订单】
  PO-001 重新显示，pending_qty=40
  
【第2次排程】
  pending_qty: 40 → 10 (排程30卷)
  
【第3次排程】
  pending_qty: 10 → 0 (排程10卷，完全排程完毕)
  
【查询待排程订单】
  PO-001 不再显示（WHERE pending_qty > 0）
```
