## 🔄 业务流程完成

### 待排程订单查询
```
【用户打开"待排程订单"tab】
           ↓
【系统自动调用 loadPendingOrders()】
           ↓
【查询条件】WHERE pending_qty > 0
           ↓
【返回结果】所有待排程的订单
           ↓
【表格显示】
  ├─ 待排数量：用彩色标签显示
  ├─ 本次排程：用纯输入框，用户可编辑
  └─ 初始值：schedule_qty = pending_qty
```

### 批排程流程
```
【用户勾选订单】
  ✓ PO-001 (pending_qty=100)
  ✓ PO-002 (pending_qty=85)
           ↓
【用户输入排程数】
  PO-001: 60 → 验证通过
  PO-002: 85 → 验证通过
           ↓
【用户点击"批量排程(2)"】
           ↓
【确认对话框】"确认排程 2 个订单，总计 145 卷？"
           ↓
【确认后】
  ├─ 前端收集数据：details[{id:1, qty:60}, {id:2, qty:85}]
  ├─ 调用 API：autoSchedule(details)
  ├─ 后端处理排程逻辑
  └─ 前端自动刷新列表
           ↓
【显示成功】
  ├─ "排程成功！已排程 2 个订单"
  ├─ 待排程订单列表刷新
  ├─ 计划管理列表刷新
  └─ 表格重新加载（pending_qty 更新）
```

## 💡 核心设计思想

### 1. 分离关注点
```
数据持久化：pending_qty （在 sales_order_items 表）
排程历史：schedule_qty （在 schedule_order_item 表）
工序任务：task_type, quantity （在 schedule_task 表）
```

### 2. 支持分批排程
```
同一订单可多次排程
每次排程从 pending_qty 中减少
排程历史完整保存
```

### 3. 数据一致性
```
创建排程 → 更新 pending_qty
多表事务 → 保证一致性
验证在前 → 防止非法数据
```
